# v0.4.2 Accepted Receipt Design-Candidate Library

v0.4.2 promotes the `採用` rows from the receipt-structure selection review into a curated synthetic receipt design-candidate library.

This release does not replace the frozen `v0.2.0` dataset payload or the `v0.4.0` Clean/Noisy leaderboard. It updates the generator side of the project so the next dataset expansion can start from a stronger, visually broader set of receipt designs.

Important wording note: the 83 accepted candidates include semantic structures, layout variants, and branding/logo/typography variants. Do not treat all 83 as audited distinct semantic structures until the taxonomy and uniqueness audit is complete.

## QA Note

`v0.4.2` is a receipt design-candidate library. It keeps all accepted candidates, including several same-family variants that still share a few item-name/quantity/amount triples. Those overlaps are not published as final benchmark records; they are explicitly deferred to the next taxonomy, uniqueness-audit, and randomized 100-type dataset generation pass.
